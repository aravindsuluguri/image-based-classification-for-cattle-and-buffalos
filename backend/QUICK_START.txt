## 🎯 Quick Start (3 Steps)

### Step 1: Install Dependencies
```bash
cd <project>/cattle\ updated/backend
pip install -r requirements_testing.txt
```

### Step 2: Organize Your Test Data

You have two options:

**Option A: If you have test images** - Organize them by class:
```bash
python prepare_test_data.py \
  --input-dir /path/to/your/test/images \
  --output-dir ./test_data \
  --metadata-path ./metadata.json
```

**Option B: If you don't have separate test images** - Use training data:
```bash
# Copy from results/
cp -r ../results/outputs ./test_data
```

### Step 3: Run Evaluation
```bash
python test_model.py
```

**OR use the quick start script:**
```bash
bash run_evaluation.sh --test-dir ./test_data
```

## ❓ FAQ

### Q: Where is my test data?
**A:** Based on your project structure, you might have images in:
- `results/outputs` (check if it's a folder with images)
- Or provide your own path using `--test-dir` flag

### Q: Can I test with just a few images?
**A:** Yes! Use the `--quick` flag or just manually copy 10-20 images to test_data:
```bash
bash run_evaluation.sh --test-dir ./test_data --quick
```

### Q: Do I need GPU?
**A:** No, but it's faster. CPU works fine:
- Edit test_model.py and set `device: 'cpu'`
- Evaluation will take longer but give same results

### Q: What if I get "No images found"?
**A:** Check:
1. Path is correct: `ls -la /your/path`
2. Images have extensions: .jpg, .jpeg, .png, .gif, .bmp
3. There are images in the directory

### Q: How long does evaluation take?
**A:** Depends on:
- Number of test images
- Image size (384x384 in your case)
- GPU/CPU speed
- Batch size

**Estimate:** ~5-30 seconds on GPU, ~1-5 minutes on CPU

## 🔧 Configuration Reference

Edit these in `test_model.py` CONFIG section:

| Setting | Default | What It Does |
|---------|---------|--------------|
| model_path | './best_model.pth' | Path to your trained model |
| metadata_path | './metadata.json' | Path to class names & config |
| test_data_dir | '../../../results/outputs' | Where test images are |
| img_size | 384 | Image size (don't change) |
| batch_size | 32 | Images processed at once (↑ faster, ↓ less memory) |
| device | 'cuda' | 'cuda' for GPU, 'cpu' for CPU |
| seed | 42 | Random seed for reproducibility |

## 🎓 Understanding the Metrics

### Accuracy
- Overall % of correct predictions
- Best for balanced datasets
- **Range:** 0-1 (higher is better)

### Precision
- "When we predict a breed, how often are we right?"
- **Range:** 0-1 (higher is better)

### Recall
- "Of real instances, how many did we catch?"
- **Range:** 0-1 (higher is better)

### F1 Score
- Harmonic mean of Precision & Recall
- Good single metric for balanced assessment
- **Range:** 0-1 (higher is better)

### G-Mean
- Geometric mean of per-class recalls
- Specially good for imbalanced multi-class data
- **Range:** 0-1 (higher is better)

### AUC-ROC
- How well the model discriminates between classes
- **Range:** 0-1 (0.5 = random, 1.0 = perfect)

## 🛠️ Troubleshooting Quick Reference

| Problem | Solution |
|---------|----------|
| `ModuleNotFoundError: torch` | `pip install -r requirements_testing.txt` |
| `CUDA out of memory` | Reduce batch_size in CONFIG (try 8 or 16) |
| `No images found` | Check path, verify images have .jpg/.png extensions |
| `Model loading fails` | Ensure best_model.pth exists, try CPU mode |
| `Permission denied` on .sh | `chmod +x run_evaluation.sh` |
| Different results each run | Set CONFIG['seed'] to consistent value |
